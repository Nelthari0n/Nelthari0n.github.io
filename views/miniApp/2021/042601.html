    # 小程序入门——项目结构

    这里并不会具体讲解很多，因为我们的项目几乎都不会是直接的原生小程序开发，根据使用的编译工具不同，实际的项目结构也不同。

    要知道一个项目都有哪些内容，首先你需要了解的就是入口文件。

    小程序项目的入口文件有三个：

    # app.js

    小程序启动的主入口文件，需要在这个文件中调用 App 方法来注册小程序实例，可以绑定小程序的生命周期回调函数； App

    # app.json

    这是小程序的公共配置文件，里面配置了小程序的页面路径、分包、导航栏的样式、tabbar的配置等等； 要注意的一点是：只有在app.json中配置的页面，才能在小程序中访问 全局配置

    # app.wxss

    公共样式表文件，内容同CSS，里面存放着一些可以在其他页面中直接使用的公共样式， 这是一个非必需文件，如果你没有任何需要写入的公共样式的话，可以没有这个文件，不过应该很少有这种情况吧。 页面文件：如果你了解 Web 开发的话，应该知道，一个页面就是一个 HTML 文件，内部可能还引用了一些 CSS 文件和 JS 文件，以及一些图片和其他文件。

    而小程序的页面由 同路径、同名称 的四个文件构成：

    js 文件存放页面逻辑，需要调用 Page 方法来注册页面实例 Page wxml 文件存放页面结构，展示内容  WXML语法参考 json 文件存放页面配置 页面配置 wxss 文件存放页面样式 WXSS

    # 小程序详细配置官方文档

    # 数据绑定

    页面 JS 文件调用 Page 方法来注册页面时，可以传入一个 data 对象，来作为页面的初始数据。

    wxml 文件渲染的内容中，动态数据都需要绑定在 JS 文件中的 data 对象上，也只有绑定在 data 中的数据，才能在 wxml 文件中得到渲染展示。【wxs后面再说】

    而数据的更新，需要在 JS 的 Page 内通过 this.setData 方法来更新具体的 data 数据，不能用类似 this.data.name = 'new name' 这样的形式直接修改，这样无法更新视图。

    提醒1：不要随便一个数据都放在 data 中，请只将需要在视图中使用的数据绑定在 data 上，其他的数据存储在其他地方。

    提醒2：在 WXML 中使用绑定数据时，可以使用对象属性访问，但是不可调用方法（WXS除外）。

    # WXS模块

    WXS 是小程序的一套脚本语言，它和 JavaScript 类似，但并不一致。

    它不依赖于运行时的基础库版本，可以在所有版本的小程序中运行。

    它的运行环境和其他 JavaScript 是隔离的，WXS 中不能调用其他 JavaScript 文件中定义的函数，也不能调用小程序提供的API。

    由于运行环境的差异，在 iOS 设备上小程序内的 WXS 会比 JavaScript 代码快 2 ~ 20 倍。

    提示：你可能会发现我们的项目中很少用到 WXS，但这并不是说它不好用，仅仅是因为我们不只是在做微信端小程序，还有支付宝小程序、百度智能小程序、字节跳动小程序……

    如此一来，我们可能会更加着重于兼容性方面来考虑；尤其是目前我们正在做多端平台统一化，更是如此。

    其他方面 你会注意到，这边并没有具体讲解小程序都有哪些API，哪些组件；因为内容有很多，这里只是一个帮助新手熟悉整体内容的简单教程，其他更多的还是依赖于你自身。

    一方面在官方文档中学习，一方面也可以在实际使用中，特定查找需要使用的API和组件，通过使用加深记忆。
