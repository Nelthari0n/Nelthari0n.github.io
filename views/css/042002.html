    iPhone全面屏系列机型底部留白适配

    嘤嘤喵喵子🍑 4/20/2021 css

    # 场景:

    iPhoneX 系列的手机在屏幕的底部有一个小黑条，在屏幕底部有内容展示或交互按钮的情况下，展示效果上会有些影响。

    为了消除这个影响，需要把底部内容提高，增加底部的留白空间。

    最初小程序端是通过机型判断，针对iPhone X机型增加样式。

    这样做的劣势是未涵盖全部机型，比如新出现的iPhone 11就无法得到有效判断。

    而iOS中有在webkit新增两个函数，可以解决这个问题。

    # 适配之前需要了解的知识

    # 安全区域

    所谓安全区域，指的是一个可视窗口范围，处于这个区域中的内容不会受到手机屏幕的圆角、齐刘海、底部小黑条所影响

    # env() 和 constant()

    iOS11 新增特性，webkit的两个 CSS 函数 env() 和 constant() 用于设定安全区域与边界的距离，目前有四个值：

    1. safe-area-inset-left: 安全区域到左边界的距离
    2. safe-area-inset-right: 安全区域到右边界的距离
    3. safe-area-inset-top: 安全区域到上边界的距离
    4. safe-area-inset-bottom: 安全区域到下边界的距离 这里我们只需要关注 safe-area-inset-bottom 这个变量，因为它对应的就是小黑条的高度（横竖屏时值不一样）。

    注: constant() 是 iOS11 最初设计的函数，但是在 iOS11.2 之后被废弃，改名为 env() 为了兼容，需要两个都使用，且顺序要先 constant 后 env。 比如:

      .footer {
    padding-bottom: calc(8px + constant(safe-area-inset-bottom)); /* 兼容 iOS11.2 以下 */
    padding-bottom: calc(8px + env(safe-area-inset-bottom));      /* 兼容 iOS11.2 及以上 */
  }

    # viewport-fit

    iOS11 新增特性，苹果公司为了适配 iPhoneX 对现有 viewport meta 标签的一个扩展，用于设置网页在可视窗口的布局方式，可设置三个值：

    1. contain: 可视窗口完全包含网页内容（左图）
    2. cover：网页内容完全覆盖可视窗口（右图）
    3. auto：默认值，跟 contain 表现一致

    # 具体适配方式

    1. 通过padding增高
      .footer {
    // ...
    padding-bottom: constant(safe-area-inset-bottom);
    padding-bottom: env(safe-area-inset-bottom);
  }

    margin-bottom: env(safe-area-inset-bottom);
